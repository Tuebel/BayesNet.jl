BayesNet.jl

Minimal implementation of a Bayesian Network which is a directed acyclic graph (DAG) where each variable conditionally depends on a set of parent variables (Wikpedia).

By convention, each node in the graph represents a variable and has a unique name associated to it. Exceptions can be made for specific node implementations, i.e. a modifier which post-processes the result of its child node. A node is defined by the following abstract type:

AbstractNode

All implementations expect the following methods are expected to be implemented or the fields of the default implementation to be available:

children(node)::Tuple{Vararg{<:AbstractNode}} = node.children
model(node)::Function = node.model
rng(node)::AbstractRNG = node.rng

For the specific behavior specify one or many of the following methods for your type:

rand_barrier(node::YourNode, variables::NamedTuple, dims...)
evaluate_barrier(node::YourNode, variables::NamedTuple)
logdensityof_barrier(node::YourNode, variables::NamedTuple)
bijector_barrier(node::YourNode, variables::NamedTuple)

Naming Convention

Naming of parent-child relationship is reversed in a Bayesian network compared to DAGs. The probability of a child variable $y$ given a parent variable $x$ is $p(y|x)$. However, node $x$ is the parent of node $y$ in the resulting graph $x → y$.

Programming is done more intuitively using the graph & node notation, thus we use parent $x → y$.
